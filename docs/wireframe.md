# Wireframe / прототип интерфейса (Mermaid)

Ниже — прототип ключевых экранов для работы с системой учета спортивных достижений.
Схема показывает основные переходы между экранами и ключевые действия пользователя.

```mermaid
flowchart TB
  %% --- Auth ---
  A[Экран: Вход\n- Email\n- Пароль\n- Кнопка «Войти»] -->|успех| B{Роль пользователя}
  A -->|ошибка| A

  %% --- Student branch ---
  B -->|Ученик| S1[Главная (Ученик)\n- краткая статистика\n- последние достижения]
  S1 --> S2[Профиль ученика\n- данные\n- класс\n- мед.группа]
  S1 --> S3[Достижения\n- фильтры\n- список записей]
  S3 --> S4[Карточка достижения\n- метрика\n- значение\n- статус\n- комментарий]
  S1 --> S5[Статистика\n- графики/таблицы\n- прогресс]

  %% --- Teacher branch ---
  B -->|Учитель| T1[Главная (Учитель)\n- классы\n- уведомления\n- быстрые действия]
  T1 --> T2[Классы\n- список классов]
  T2 --> T3[Класс: список учеников\n- поиск/фильтр]
  T3 --> T4[Профиль ученика (для учителя)\n- достижения\n- статистика]
  T4 --> T5[Форма «Внести достижение»\n- метрика\n- значение\n- дата\n- комментарий\n- «Сохранить»]
  T4 --> T6[Подтверждение записей\n- список SUBMITTED\n- approve/reject]
  T1 --> T7[Отчеты\n- период\n- класс/ученик\n- экспорт PDF/Excel]

  %% --- Admin branch ---
  B -->|Администратор| A1[Панель администратора]
  A1 --> A2[Пользователи\n- создать\n- роли\n- блокировка]
  A1 --> A3[Справочники\n- виды спорта\n- метрики\n- нормативы]
  A1 --> A4[Аудит\n- журнал действий]
  A1 --> A5[Резервное копирование\n- экспорт/импорт]
```

## Комментарии к прототипу
- **Ролевая главная страница**: после входа пользователь попадает на дашборд, зависящий от роли (ученик/учитель/администратор).
- **Форма внесения достижения (учитель)**: ключевой экран для фиксации результата; после сохранения запись может получить статус `SUBMITTED`.
- **Подтверждение записей**: учитель может массово подтверждать/отклонять результаты, влияя на статус записи.
- **Отчеты**: формирование отчета по периоду с выбором класса/ученика и экспортом.
