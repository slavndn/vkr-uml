# Wireframe / прототип интерфейса (Mermaid)

Ниже — прототип ключевых экранов для работы с системой учета спортивных достижений.
Схема показывает основные переходы между экранами и ключевые действия пользователя.

```mermaid
flowchart TB
  %% --- Auth ---
  A[Экран: Вход (Email, Пароль, Войти)] -->|успех| B{Роль пользователя}
  A -->|ошибка|A

  %% --- Student branch ---
  B -->|Ученик| S1[Главная (ученик): статистика, последние достижения]
  S1 --> S2[Профиль ученика: данные, класс, мед.группа]
  S1 --> S3[Достижения: фильтры, список записей]
  S3 --> S4[Карточка достижения: метрика, значение, статус, комментарий]
  S1 --> S5[Статистика: графики/таблицы, прогресс]

  %% --- Teacher branch ---
  B -->|Учитель| T1[Главная (учитель): классы, уведомления, быстрые действия]
  T1 --> T2[Классы: список классов]
  T2 --> T3[Класс: список учеников, поиск/фильтр]
  T3 --> T4[Профиль ученика (учитель): достижения, статистика]
  T4 --> T5[Внести достижение: метрика, значение, дата, комментарий, сохранить]
  T4 --> T6[Подтверждение записей: список SUBMITTED, approve/reject]
  T1 --> T7[Отчеты: период, класс/ученик, экспорт]
```

## Комментарии к прототипу
- **Ролевая главная страница**: после входа пользователь попадает на дашборд, зависящий от роли (ученик/учитель).
- **Форма внесения достижения (учитель)**: ключевой экран для фиксации результата; после сохранения запись может получить статус `SUBMITTED`.
- **Подтверждение записей**: учитель может массово подтверждать/отклонять результаты, влияя на статус записи.
- **Отчеты**: формирование отчета по периоду с выбором класса/ученика и экспортом.
