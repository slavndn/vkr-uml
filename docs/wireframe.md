# Wireframe / прототип интерфейса (Mermaid)

Ниже — прототип ключевых экранов для работы с системой учета спортивных достижений.
Схема показывает основные переходы между экранами и ключевые действия пользователя.

```mermaid
flowchart TB
  %% --- Auth ---
  A[Экран: Вход<br/>Email<br/>Пароль<br/>Кнопка «Войти»] -->|успех|B{Роль пользователя}
  A -->|ошибка|A

  %% --- Student branch ---
  B -->|Ученик|S1[Главная (Ученик)<br/>краткая статистика<br/>последние достижения]
  S1 --> S2[Профиль ученика<br/>данные<br/>класс<br/>мед.группа]
  S1 --> S3[Достижения<br/>фильтры<br/>список записей]
  S3 --> S4[Карточка достижения<br/>метрика<br/>значение<br/>статус<br/>комментарий]
  S1 --> S5[Статистика<br/>графики/таблицы<br/>прогресс]

  %% --- Teacher branch ---
  B -->|Учитель|T1[Главная (Учитель)<br/>классы<br/>уведомления<br/>быстрые действия]
  T1 --> T2[Классы<br/>список классов]
  T2 --> T3[Класс: список учеников<br/>поиск/фильтр]
  T3 --> T4[Профиль ученика (для учителя)<br/>достижения<br/>статистика]
  T4 --> T5[Форма «Внести достижение»<br/>метрика<br/>значение<br/>дата<br/>комментарий<br/>«Сохранить»]
  T4 --> T6[Подтверждение записей<br/>список SUBMITTED<br/>approve/reject]
  T1 --> T7[Отчеты<br/>период<br/>класс/ученик<br/>экспорт PDF/Excel]
```

## Комментарии к прототипу
- **Ролевая главная страница**: после входа пользователь попадает на дашборд, зависящий от роли (ученик/учитель).
- **Форма внесения достижения (учитель)**: ключевой экран для фиксации результата; после сохранения запись может получить статус `SUBMITTED`.
- **Подтверждение записей**: учитель может массово подтверждать/отклонять результаты, влияя на статус записи.
- **Отчеты**: формирование отчета по периоду с выбором класса/ученика и экспортом.
