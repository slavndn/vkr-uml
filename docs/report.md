# Отчет по заданию: UML‑диаграммы и прототип интерфейса (ВКР)

## 1) Предметная область
Разрабатываемый продукт: **веб‑приложение для учета спортивных достижений обучающихся на уроках физической культуры в школе**.

Цель системы — упростить фиксацию результатов (нормативы/тесты/соревнования), хранение истории достижений, формирование статистики и отчетов для учителя и администрации, а также предоставить ученику доступ к собственным результатам.

Роли пользователей:
- **Ученик**: просмотр профиля, достижений и статистики.
- **Учитель физической культуры**: ведение классов/учеников, внесение и подтверждение достижений, формирование отчетов.

Особенность реализации (прототип/учебная версия):
- данные хранятся **локально в браузере (localStorage)**;
- серверная БД не используется;
- перенос данных выполняется через **импорт/экспорт JSON**.

## 3) Генерация программного кода из диаграммы классов
Код (TypeScript) сгенерирован по структуре диаграммы классов и размещен в `src/domain/`:
- `src/domain/types.ts` — типы (`UUID`, `ISODate`, `ISODateTime`, …)
- `src/domain/enums.ts` — перечисления (`Role`, `RecordStatus`)
- `src/domain/models.ts` — интерфейсы доменных сущностей
- `src/domain/index.ts` — реэкспорт

Этот код может использоваться как основа для:
- DTO/моделей backend (например, Node.js/NestJS)
- типизации frontend (React/Next.js)
- описания контрактов API

## 4) Прототип интерфейса (wireframe)
Прототип размещен в `docs/wireframe.md` и выполнен в виде схемы переходов и экранов в формате Mermaid.

## 5) Как воспроизводилось построение
1. Диаграммы описаны в текстовом виде на PlantUML (`.puml`).
2. В IDE (VS Code/JetBrains) используется предпросмотр PlantUML (расширение/плагин) для генерации изображения диаграмм.
3. По диаграмме классов подготовлены доменные модели TypeScript в `src/domain/` (структура соответствует сущностям и атрибутам).
